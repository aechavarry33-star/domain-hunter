Google Places API

Necesitas habilitar Places API (New) en Google Cloud.

1. Ve a console.cloud.google.com
2. Crea un proyecto → Credenciales → Crear API Key
3. Habilita: Places API y Geocoding API
4. (Opcional) Restringe la key a tu dominio

El tier gratuito incluye $200 USD/mes de crédito (~40,000 búsquedas).

WhoisXML API

Obtén 500 consultas gratis al mes (sin tarjeta).

1. Regístrate en whoisxmlapi.com
2. Ve a My Products → WHOIS API
3. Copia tu API Key

Endpoint usado: whoisxmlapi.com/whoisserver/WhoisService
Soporta +1,500 TLDs incluyendo .mx, .com.mx

Modo de operación

En modo Demo, Claude genera negocios realistas para tu zona y simula datos WHOIS para probar la herramienta.
En modo Real, se usan las APIs de Google Places y WhoisXML para obtener datos reales en tiempo real.

¿Para qué sirve esta herramienta?

DomainHunter busca negocios locales en Google Maps, extrae sus sitios web y verifica si los dominios están vencidos, por vencer o vigentes.

Los dominios vencidos de negocios activos son oportunidades: puedes registrarlos y ofrecerlos al negocio, o monitorear si los van a renovar.

Flujo de datos
1. Google Places Text Search → busca negocios por tipo + ciudad
2. Google Place Details → extrae el campo website de cada negocio
3. Extracción del dominio del URL (elimina https://, www, paths)
4. WhoisXML API → consulta WHOIS real → extrae expiresDate
5. Clasificación: Vencido / <60 días / Vigente
Costos estimados
Google Places Text Search: $0.017 USD / request (1 búsqueda por sesión)
Google Place Details: $0.017 USD / place (por cada negocio encontrado)
WhoisXML API: 500 gratis/mes → ~$0.001 USD/query después
→ Buscar 10 negocios ≈ $0.19 USD total
→ Google da $200 USD gratis/mes = ~1,000 búsquedas de 10 negocios
